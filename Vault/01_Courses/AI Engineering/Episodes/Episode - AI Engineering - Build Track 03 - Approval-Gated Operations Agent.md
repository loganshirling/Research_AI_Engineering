# Episode - AI Engineering - Build Track 03 - Approval-Gated Operations Agent

## Goal

Track the teaching artifact for the third build-track capstone: an approval-gated operations agent.

## Source note

- [[Practice - End-to-End Build - Approval-Gated Operations Agent]]

## Intended learning outcome

After this episode, the learner should be able to explain:
- why action-taking agents are application-control problems, not just prompting problems,
- how approval checkpoints should be designed and logged,
- what must remain deterministic in an operations harness,
- and how safe execution differs from merely producing a plausible plan.

## Notes

This episode should emphasize bounded action, explicit approvals, operator visibility, and rollback thinking.

It should make clear that the goal is not maximal autonomy.
The goal is trustworthy execution under controlled permissions.
