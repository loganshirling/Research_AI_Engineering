## Use this note to check course readiness

### Foundations
- Can you explain tokens, embeddings, attention, context windows, prefill, and decode in practical language?
- Can you connect those ideas to cost, latency, and memory instead of describing them only academically?

### Retrieval
- Can you sketch the full retrieval pipeline?
- Can you diagnose chunking, metadata, search-mode, reranking, and context-packing failures separately?

### Model choice and evals
- Can you define the eval before changing the system?
- Can you compare models using task fit, latency, cost, format reliability, and safety behavior?

### Safety and observability
- Can you explain why prompts are not a security boundary?
- Can you say what traces, logs, and regression gates must exist for a production system?

### Advanced systems
- Can you justify where the system should stay deterministic?
- Can you explain when approvals are required?
- Can you separate text retrieval, page understanding, extraction, and downstream action-taking?

### Build readiness
- Can you write a one-page architecture memo for:
  - an internal research assistant
  - a multimodal document operations copilot
  - an approval-gated operations agent
